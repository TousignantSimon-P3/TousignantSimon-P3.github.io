    Les Fichiers Texte en C++

    La manipulation de fichiers texte est essentielle pour la persistance de données lisibles par l'humain.

    1. Flux Standards

    En C++, on utilise la bibliothèque <fstream> :
    std::ofstream : Pour l'écriture (Output File Stream).
     std::ifstream : Pour la lecture (Input File Stream).
    * std::fstream : Pour les deux.

    2. Encodage et Unicode

    • UTF-8 : C'est la norme moderne. En C++, un std::string peut stocker de l'UTF-8.
    • Affichage : Attention, la console Windows (cmd) nécessite souvent une configuration spécifique (chcp 65001) pour afficher correctement les caractères accentués.

    3. Portabilité et Fins de ligne (r, n)

    Les systèmes d'exploitation ne s'accordent pas sur la façon de marquer la fin d'une ligne.

    • LF (Line Feed) : \n. Valeur ASCII 10 (0x0A). Utilisé par Linux et macOS.
    • CR (Carriage Return) : \r. Valeur ASCII 13 (0x0D). Utilisé historiquement par les vieux Mac.
    • CRLF : \r\n. Suite des deux caractères (13 puis 10). Utilisé par Windows.

    Pourquoi est-ce important ?

    Si vous écrivez un fichier sur Windows et l'ouvrez sur Linux, certains éditeurs afficheront des caractères ^M à la fin de chaque ligne (représentant le \r). Inversement, un fichier Linux ouvert dans de vieux outils Windows s'afficherait sur une seule ligne.

    Le standard universel : Aujourd'hui, la majorité des outils (Git, VS Code, etc.) préfèrent le format LF (\n) et l'encodage UTF-8.

    4. Encodage : std::string vs std::wstring

    Pour qu'un fichier soit "universel", il doit utiliser un encodage compris par tous les systèmes.

    • UTF-8 (Standard recommandé) :
      • Utilise std::string.
      • Compatible avec l'ASCII (les 127 premiers caractères sont identiques).
      • C'est le format universel du Web et de la majorité des systèmes modernes.
    • Gestion des caractères spéciaux (é, è, à, Arabe, etc.) :
      • En UTF-8, un caractère peut occuper plus d'un octet dans un std::string. Par exemple, 'é' occupe 2 octets, alors qu'un caractère arabe comme 'ش' peut en occuper 2 ou 3.
      • Attention : maString.length() renvoie le nombre d'octets, pas le nombre de caractères visibles.
      • Le dilemme (La chèvre et le chou) : Comment avoir l'universalité de l'UTF-8 tout en sachant combien il y a de caractères ?
        • En UTF-8, les octets de "continuation" commencent par les bits 10. On peut donc compter uniquement les octets qui ne sont pas des continuations pour obtenir le nombre de points de code (caractères).
          #include <algorithm>  

size_t compterCaracteres(const std::string& s) {  
    return std::count_if(s.begin(), s.end(), [](unsigned char c) {  
        return (c & 0xC0) != 0x80; // On ignore les octets de continuation  
    });  
}

      • Bibliothèques spécialisées : Pour des projets plus complexes, on évite de réinventer la roue :

        • utf8cpp (GitHub : nemtrif/utfcpp) : Une bibliothèque légère (header-only) très populaire.

        Exemple avec utf8cpp :

        #include "utf8.h"  
#include <string>  
#include <iostream>  

void exempleUTF8() {  
    std::string s = "Héllò ش"; // UTF-8  

    // 1. Compter les vrais caractères (distance)  
    size_t nb = utf8::distance(s.begin(), s.end());  

    // 2. Vérifier si la chaîne est valide  
    bool valide = utf8::is_valid(s.begin(), s.end());  

    // 3. Parcourir les points de code  
    auto it = s.begin();  
    while (it != s.end()) {  
        uint32_t cp = utf8::next(it, s.end());  
        // cp contient la valeur numérique du caractère (ex: 0x00E9 pour 'é')  
    }  
}
        * ICU : Le standard professionnel absolu, mais très lourd à intégrer.
        * Affichage Windows : Pour voir ces caractères en console, il faut forcer l'encodage UTF-8 au début du programme :
        #ifdef _WIN32  
system("chcp 65001 > nul"); // Force la console Windows en UTF-8  
#endif
        * UTF-16 / wstring :
        * Utilise std::wstring et wchar_t.
        * Problème de portabilité : La taille de wchar_t change selon le système (2 octets sur Windows, 4 octets sur Linux).
        * À éviter pour le stockage de fichiers si vous visez l'interopérabilité.

    Conseil : Utilisez toujours std::string avec de l'UTF-8 pour garantir que vos fichiers soient lisibles partout sans conversion complexe.

    5. Exemple de lecture sécurisée

    #include <iostream>  
#include <fstream>  
#include <string>  

void lireTexte(const std::string& chemin) {  
    std::ifstream fichier(chemin);  
    if (fichier.is_open()) {  
        std::string ligne;  
        while (std::getline(fichier, ligne)) {  
            std::cout << ligne << std::endl;  
        }  
    }  
}
