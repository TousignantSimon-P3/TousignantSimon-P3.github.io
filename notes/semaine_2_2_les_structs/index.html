    De la struct à la class : Une Introduction Progressive

    L'objectif de cette note est de comprendre l'évolution de la pensée en C++ : comment on passe d'une simple structure de données à un objet complet et robuste. Nous allons suivre une progression en plusieurs étapes.

    Partie 1 : La struct comme Simple Conteneur de Données

    Au niveau le plus simple, une struct est une manière de regrouper plusieurs variables en un seul type. On la définit généralement dans un fichier d'en-tête (.hpp) pour pouvoir la réutiliser.

    1.1 - L'En-tête (Joueur.hpp) et les Gardes

    Un fichier d'en-tête doit être protégé contre l'inclusion multiple. Si un même .hpp est inclus plusieurs fois, le compilateur verra la définition de la struct en double et générera une erreur. Pour éviter cela, on utilise une "garde d'en-tête".

    Méthode Moderne (#pragma once) : À placer sur la toute première ligne. C'est la méthode privilégiée pour ce cours.

    // Fichier : Joueur.hpp
#pragma once
#include <string>

// La seule responsabilité de ce fichier est de définir la struct Joueur.
struct Joueur {
    std::string nom;
    int points_de_vie;
    int niveau;
};

    Méthode Traditionnelle (ifndef) : Fonctionne sur tous les compilateurs. Le nom (JOUEUR_HPP) doit être unique.

    #ifndef JOUEUR_HPP
#define JOUEUR_HPP
// ... contenu du .hpp ...
#endif // JOUEUR_HPP

    1.2 - Utilisation simple dans main.cpp

    Le main.cpp inclut le .hpp et peut ensuite créer et manipuler des variables du type Joueur.

    // Fichier : main.cpp
#include <iostream>
#include <format>
#include "Joueur.hpp" // On inclut la définition de notre struct

int main() {
    Joueur p1 {
        .nom = "Frodon Sacquet",
        .points_de_vie = 20,
        .niveau = 1
    };

    p1.niveau = 2; // Accès direct aux membres publics

    std::cout << std::format("{} est maintenant niveau {}\n", p1.nom, p1.niveau);
    return 0;
}

    Partie 2 : struct avec Fonctions Libres

    Rapidement, on se rend compte que du code se répète (par exemple, l'affichage d'un joueur). On peut créer des fonctions pour ces tâches. Pour garder notre projet organisé, on sépare les déclarations des définitions.

    • JoueurActions.hpp : Déclare les fonctions "libres" qui travaillent sur Joueur. C'est le "menu des actions".
    • JoueurActions.cpp : Implémente le code de ces fonctions.

    // Fichier : JoueurActions.hpp
#pragma once
#include "Joueur.hpp" // Pour savoir ce qu'est un Joueur

void afficher_joueur(const Joueur& j); // Signature (prototype)

    // Fichier : JoueurActions.cpp
#include "JoueurActions.hpp"
#include <iostream>
#include <format>

void afficher_joueur(const Joueur& j) { // Implémentation
    std::cout << std::format("- Nom: {}, PV: {}\n", j.nom, j.points_de_vie);
}

    // Fichier : main.cpp (modifié)
#include "JoueurActions.hpp" // On inclut maintenant les actions

int main() {
    Joueur p1 { .nom = "Sam Gamegie", .points_de_vie = 30 };
    afficher_joueur(p1); // On utilise notre fonction
    return 0;
}

    Partie 2.1 : Structure, Inclusions et Compilation

    Cette séparation des fichiers suit un principe essentiel : le Principe de Responsabilité Unique. Chaque fichier a un seul rôle.

    • Joueur.hpp : Définit ce qu'EST un Joueur.
    • JoueurActions.hpp : Déclare ce qu'on PEUT FAIRE avec un Joueur.
    • JoueurActions.cpp : Explique COMMENT on le fait.
    • main.cpp : Lance le programme.

    Schéma des Inclusions

    Voici comment les fichiers dépendent les uns des autres. Une flèche A -> B signifie que le fichier A inclut le fichier B.

               main.cpp  ---------------> JoueurActions.hpp
                                          |
JoueurActions.cpp  ------------> JoueurActions.hpp
                                          |
                                          v
                                      Joueur.hpp

    Le main.cpp n'a pas besoin de connaître les détails de Joueur.hpp. Il fait confiance à JoueurActions.hpp pour lui fournir les actions nécessaires.

    Compilation du Projet

    Pour créer l'exécutable, vous devez fournir tous les fichiers .cpp au compilateur. Celui-ci va les traduire en code machine, puis l'éditeur de liens (linker) va les assembler en un seul programme.

    g++ -std=c++20 -Wall -o mon_programme main.cpp JoueurActions.cpp
    Notez qu'on ne donne jamais les fichiers .hpp au compilateur directement. Ils sont inclus par les .cpp.

    Partie 3 : Évolution vers les Fonctions Membres

    L'étape suivante est de lier plus fortement un comportement à la struct elle-même. Si une fonction est intrinsèquement liée à un objet (comme afficher), on peut en faire une fonction membre.

    Important : À ce stade, la struct reste un conteneur "passif" avec des données publiques. C'est une étape d'organisation, pas encore de protection.

    • Joueur.hpp : La fonction est déclarée DANS la struct.
    • Joueur.cpp : Un fichier source pour Joueur implémente ses fonctions membres.

    // Fichier : Joueur.hpp (refactorisé)
#pragma once
#include <string>

struct Joueur {
    std::string nom;
    int points_de_vie;
    int niveau;
    void afficher() const; // Déclaration de la fonction MEMBRE
};

    // Fichier : Joueur.cpp (nouveau)
#include "Joueur.hpp"
#include <iostream>
#include <format>

void Joueur::afficher() const { // Joueur:: indique que c'est une fonction de Joueur
    std::cout << std::format("- Nom: {}, PV: {}\n", nom, points_de_vie);
}

    // Fichier : main.cpp (refactorisé)
#include "Joueur.hpp"

int main() {
    Joueur p1 { .nom = "Gandalf", .points_de_vie = 1000 };
    p1.afficher(); // Appel plus naturel : on demande à l'objet de s'afficher.
    return 0;
}

    Partie 4 : Gestion de la Mémoire et Fonctions

    Comment passer et retourner des structs efficacement et sans danger.

    4.1 - Accès : Point (.) vs. Flèche (->)

    Si vous avez... Opérateur Syntaxe Exemple
    L'objet ou une référence Le point (.) mon_objet.membre p1.nom
    Un pointeur vers l'objet La flèche (->) ptr_objet->membre ptr_p1->nom

    La flèche -> est un raccourci pour déréférencer puis accéder : (*ptr_p1).nom.

    4.2 - Passage en Paramètre

    • Lecture Seule (const&) : Le plus sûr et le plus efficace pour passer des objets que la fonction ne doit pas modifier. Évite une copie. 
      void afficher_details(const Joueur& j) { /* ... */ }
    • Lecture/Écriture (&) : Quand la fonction DOIT modifier l'objet original. 
      void monter_niveau(Joueur& j) { j.niveau += 1; }

    4.3 - Retour de struct depuis une fonction

    • Par Copie (Sûr) : La fonction retourne une copie. L'objet original créé dans la fonction est détruit, mais sa copie survit. 
      Joueur creer_ennemi() {
    Joueur orc { .nom = "Orc de base", .niveau = 1 };
    return orc; // OK : une copie de 'orc' est envoyée.
}
    • Par Pointeur (DANGEREUX !) !!! failure "À NE JAMAIS FAIRE" Il ne faut jamais retourner l'adresse (&) ou un pointeur (*) d'une variable locale. La variable est détruite à la fin de la fonction, et le pointeur devient un "dangling pointer" qui pointe vers une zone mémoire invalide. Utiliser ce pointeur mène à un comportement indéfini (crash, corruption). 
      Joueur* creer_ennemi_dangereux() {
    Joueur gobelin { .nom = "Gobelin", .niveau = 1 };
    return &gobelin; // ERREUR FATALE !  
    /* 
     * gobelin va être détruit après la fonction.  
     * C'est une variable locale, donc, le pointeur  
     * va pointer sur une valeur instable.  
     * (mémoire libre pour autre chose) 
     */
}

    Partie 5 : Ouverture vers la class (Semaine 3)

    Le problème de nos structs actuelles est que leurs données sont publiques et vulnérables (p1.niveau = -99;). La prochaine étape est d'apprendre à protéger les données et à garantir des règles (invariants) en utilisant le mot-clé class.

    Aperçu de l'Évolution en class

    // Fichier : Joueur.hpp (Version Semaine 3)
#pragma once
#include <string>

class Joueur {
private: // Les données sont maintenant protégées et inaccessibles de l'extérieur.
    std::string nom_;
    int points_de_vie_;
    int niveau_;

public:
    // Un "constructeur" pour garantir que chaque Joueur est créé dans un état valide.
    Joueur(const std::string& nom, int pv, int niveau);

    // Une fonction membre pour une modification contrôlée des données.
    void subir_degats(int degats);

    // Une fonction membre pour un accès en lecture seule.
    void afficher() const;

    // Un "getter" pour lire une valeur privée de manière contrôlée.
    int get_niveau() const;
};
    Cette structure est le cœur de la programmation orientée objet en C++ et sera l'objet de la semaine 3.

    Exercice 2 : Fonctions Libres

    1. Créez LivreActions.hpp/.cpp.

    Exercice 3 : Fonctions Membres

    1. Modifiez Livre.hpp pour y déclarer une fonction membre void afficher() const;.
